## 📖 نبذة عن المشروع
**Fatwa CMS Bot** ليس مجرد بوت بحث تقليدي، بل هو نظام متكامل لإدارة المحتوى (CMS) مصمم خصيصاً للمؤسسات الدعوية وطلبة العلم. يتميز النظام بالقدرة على أرشفة آلاف الفتاوى وتقديمها للمستخدمين بأساليب بحث ذكية وتصنيفات فقهية دقيقة، مع دعم كامل للوسائط المتعددة والجدولة الآلية.

> [!IMPORTANT]
> **قاعدة بيانات ضخمة:** يأتي النظام مجهزاً بقاعدة بيانات تضم آلاف الفتاوى المفرغة والمؤرشفة لكبار علماء أهل السنة والجماعة، مع تتبع دقيق للمصادر والروابط الصوتية.

## 🚀 المميزات التقنية الرئيسية

### 🔍 1. محرك البحث الذكي (AI & Semantic Search)
- **البحث الدلالي:** معالجة متقدمة للغة العربية تشمل (إزالة التشكيل، توحيد الهمزات، والتعامل مع زوائد الكلمات).
- **تقنية RAG (Retrieval-Augmented Generation):** يقوم البوت بتحليل سؤال المستخدم عبر الذكاء الاصطناعي (**Groq/Llama 3.3**)، استخراج الفتاوى ذات الصلة، ثم صياغة إجابة ملخصة مع الاستشهاد بالمصادر الأصلية.
- **البحث المتقدم (Smart Filters):** إمكانية التصفية حسب (اسم العالم، التصنيف الفقهي، نوع المصدر، أو رقم الفتوى).

### 📢 2. نظام النشر والانتشار المتطور
- **النشر التلقائي (Auto-Publish):** جدولة آلية لنشر "فتوى اليوم" في القنوات والمجموعات المشتركة.
- **نشر المستخدمين (User-Led Publishing):** ميزة فريدة تتيح للمستخدمين اختيار فتاوى من تصنيفات محددة ونشرها في قنواتهم الخاصة التي يشرف عليها البوت.
- **التقارير الأسبوعية:** إرسال ملخصات دورية بأبرز الفتاوى والدروس للمشتركين.

### 🛠️ 3. لوحة تحكم إدارية شاملة (Admin Dashboard)
- **إدارة المحتوى (CRUD):** إضافة وتعديل وحذف الفتاوى، العلماء، التصنيفات، والمصادر بواجهة تفاعلية سهلة.
- **إدارة القنوات:** تتبع حالة البوت في القنوات والمجموعات، مع ميزة "الحذف من الكل" لسحب الفتاوى في حال الخطأ.
- **إدارة المسؤولين:** نظام مستويات للوصول والتحكم في إعدادات البوت الحساسة.
- **وضع الصيانة:** إمكانية قفل البوت للصيانة مع رسالة مخصصة للمستخدمين.

### 🛡️ 4. الاستقرار والأداء
- **Singleton Lock:** نظام حماية يمنع تشغيل أكثر من نسخة للبوت في آن واحد لضمان سلامة قاعدة البيانات.
- **Rate Limiting:** حماية ضد الإغراق (Spam) وضمان استقرار الخدمة لجميع المستخدمين.
- **Async Architecture:** كود برمجي غير متزامن بالكامل يعتمد على `aiosqlite` لضمان سرعة الاستجابة تحت ضغط الطلبات العالية.

## 📂 هيكلية المشروع (Architecture)

```text
├── main.py              # نقطة انطلاق البوت وتهيئة الجدولة والمهام
├── core/                # النواة الأساسية (قاعدة البيانات، الإعدادات، الأدوات)
│   ├── database.py      # مدير قاعدة بيانات الفتاوى (FTS5 Support)
│   ├── bot_db.py        # مدير بيانات المستخدمين والإعدادات العامة
│   └── utils.py         # الدوال المساعدة ومعالجة النصوص والأمان
├── handlers/            # معالجات الأوامر والتفاعل (Modular Handlers)
│   ├── admin/           # لوحة الإدارة بكافة أقسامها
│   ├── channels/        # تتبع القنوات والنشر التلقائي والمهام الخلفية
│   ├── fatwa/           # إضافة، تعديل، وعرض الفتاوى (Logic & UI)
│   ├── search/          # محرك البحث (الذكي، المتقدم، والتصفح)
│   └── user_publish.py  # منطق نشر المستخدمين في قنواتهم الخاصة
├── data/                # قواعد البيانات والنسخ الاحتياطية
└── logs/                # سجلات العمليات (Log Rotation enabled)
```

## 🛠️ التثبيت والتشغيل

### المتطلبات الأساسية
- **Python:** 3.10 or higher
- **Telegram Token:** من [@BotFather](https://t.me/BotFather)
- **Groq API Key:** (اختياري) من [Groq Console](https://console.groq.com/)

### خطوات التشغيل
1. **تحميل المشروع:**
   ```bash
   git clone https://github.com/your-username/fatwa-cms-bot.git
   cd fatwa-cms-bot
   ```

2. **تهيئة البيئة الافتراضية:**
   ```bash
   python -m venv venv
   source venv/bin/activate  # Linux/macOS
   .\venv\Scripts\activate   # Windows
   ```

3. **تثبيت التبعيات:**
   ```bash
   pip install -r requirements.txt
   ```

4. **إعداد المتغيرات البيئية:**
   قم بنسخ ملف `.env.example` إلى `.env` واملأ البيانات المطلوبة:
   ```env
   TELEGRAM_TOKEN=your_token_here
   OWNER_ID=your_id_here
   GROQ_API_KEY=your_key_here
   ```

5. **تشغيل البوت:**
   ```bash
   python main.py
   ```
